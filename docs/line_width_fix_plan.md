# 계획_1: Three.js 선 굵기(LineWidth) 제한 해결 및 시각적 업그레이드

## 1. 문제 분석 (Problem Definition)
현재 `charmony-3d` 프로젝트 및 `threejs-projects`의 튜토리얼 코드들은 Three.js의 기본 `LineBasicMaterial`을 사용하고 있습니다. WebGL의 기술적 제약으로 인해 대부분의 브라우저에서 이 방식은 **선의 굵기를 1픽셀 이상으로 표현할 수 없습니다.**

이로 인해 발생하는 시각적 한계는 다음과 같습니다:
- **미적 품질 저하**: 고해상도 화면에서 선이 너무 가늘어 보여 시인성이 떨어짐.
- **예술적 표현 제한**: `maze-art`와 같은 강렬한 선 표현이 불가능함.
- **물리적 존재감 부족**: Beam Collider 등의 상호작용에서 빔의 세기나 두께를 표현하기 어려움.

## 2. 해결 방안 (Proposed Solution)
기본 `THREE.Line` 대신, `@react-three/drei` 라이브러리의 **`<Line>` 컴포넌트(Fat Lines 기반)**를 도입합니다. 내부적으로 `Line2`와 `LineMaterial`을 사용하므로, 성능 손실 없이 자유로운 선 굵기 조절이 가능해집니다.

### 핵심 이점:
- **실제 선 굵기 조절**: `lineWidth` 속성을 통해 브라우저 제약 없이 굵기 조절 가능.
- **해상도 독립성**: 화면 해상도나 카메라 거리에 따라 굵기를 고정하거나 가변적으로 설정 가능.
- **그라데이션 지원**: 선의 시작과 끝에 서로 다른 색상을 입히는 기능(Vertex Colors)을 완벽하게 지원.

## 3. 세부 구현 계획 (Implementation Steps)

### 3.1. `LissajousCurve` 컴포넌트 업그레이드
- `THREE.Line`을 `<Line>`(drei)으로 교체.
- 사용자가 UI 슬라이더를 통해 선의 굵기를 실시간으로 조절할 수 있도록 `lineWidth` 속성 연결.

### 3.2. `LorenzAttractor` 컴포넌트 업그레이드
- 카오스 시뮬레이션의 궤적을 굵은 선으로 표현하여 몰입감 증대.
- `Bloom` 효과와 연동하여 네온 느낌의 강렬한 레이저 선 구현.

### 3.3. `BeamCollider3D` 컴포넌트 업그레이드
- `rayWidth` 파라미터를 실제 `<Line>`의 `lineWidth`와 동기화.
- 반사(Bounce) 횟수에 따라 선이 가늘어지거나 옅어지는 효과 추가 가능.

## 4. 기대 효과 (Expected Results)
- **WOW 포인트**: 1픽셀의 밋밋한 선에서 벗어나, `maze-art` 수준의 화려하고 임팩트 있는 시각적 경험 제공.
- **사용자 커스터마이징**: 사용자가 직접 선의 굵기를 조절하며 시각적 재미를 느낄 수 있음.
