# 8_sort-3d Status

Date: 2026-04-03

## Summary

`8_sort-3d`는 `7_sort-bar`에서 가져온 정렬 알고리즘을 사용하는 독립 실험 프로젝트입니다.
현재는 알고리즘 모듈 일부만 `7_sort-bar` 최신 수정분으로 반영된 상태입니다.

## Reflected Algorithm Fixes

The following files were copied from `7_sort-bar/algorithm/` into `8_sort-3d/algorithm/` on 2026-04-03:

- `selectionSort.js`
- `bitonicSort.js`
- `timSort.js`
- `strandSort.js`
- `gravitySort.js`

## Why These Files Were Updated

- `selectionSort.js`
  - Added phase events so long scans do not look like a restart from the beginning.
- `bitonicSort.js`
  - Stopped mutating the original array length with padding values.
  - Uses a working copy for power-of-two padding and writes back only the real sorted values.
- `timSort.js`
  - Added `write` events for insertion and merge stages.
  - Added phase events so merge progress is visible in the HUD.
- `strandSort.js`
  - Stopped shrinking the original array during sorting.
  - Uses a working input copy and writes sorted output back to the original array incrementally.
- `gravitySort.js`
  - Added phase/write/focus events so the algorithm visibly starts and progresses.

## Known Scope Gap

Only algorithm files were reflected into `8_sort-3d`.

`7_sort-bar` changes should be used as reference only.
Do not apply `7_sort-bar` controller/UI behavior to `8_sort-3d` blindly.
`8_sort-3d` has its own interaction model and should be updated intentionally, case by case.

The following non-algorithm fixes from `7_sort-bar` have not been reflected into `8_sort-3d` yet:

- controller-level behavior changes
  - all-mode stop behavior
  - timer reset/accumulation behavior tuning
- visualizer-level behavior changes
  - indicator mesh reuse
  - Three.js resource disposal for highlight indicators

If `8_sort-3d` shows similar UI/performance issues, compare and port the equivalent controller/renderer files next.
